这是一个以排障、定位、还原现场为核心目标的调试工具库。它的第一价值不是“展示数据”，而是“提供真实、可信、可依赖的调试时间线”。

日志、网络、错误以及任何可观测事件，都必须尽可能严格按照事件真实触发时间排序与呈现。真实触发顺序不能被任何实现细节所扭曲。只要可能误导开发者对因果关系和时间线的判断，就属于严重缺陷。

在这个准确性底线之上，必须想尽一切办法把性能与资源开销做到极致。可以允许一切发散性思维和非常规设计，只要最终能更高效地达成目标。思考方式不要停留在 JS 层或普通 UI 组件层，而要尽量从 Android / iOS 底层机制出发，自下而上设计这个工具库

目标不是做一个“功能差不多能用”的调试面板，而是做一个在真实复杂问题下依然可靠、准确、低开销、反应快、可长期依赖的高性能底层调试工具库。

决策优先级如下：
1. 准确性与时间线真实性不能被牺牲
2. 在不牺牲准确性的前提下，把性能与资源效率压榨到极致
3. 为达成以上目标，鼓励底层化思考、系统级思考、跨层协同优化，而不是局限于表层修补
4. 任何“为了性能而接受错误顺序、关键日志缺失、关键事件漏记”的方案，默认不可接受
